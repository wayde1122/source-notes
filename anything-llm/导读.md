---

## 一句话总结

一个**全栈 RAG 应用**：React 前端 + Express 后端 + 独立文档处理服务，支持十几种 LLM 和向量数据库的热插拔。

---

## 项目结构

```
anything-llm/
├── server/              # 主后端：Express + Prisma + RAG 核心
├── frontend/            # React + Vite 前端
├── collector/           # 独立文档处理服务（端口 8888）
├── docker/              # Docker 部署配置
├── cloud-deployments/   # AWS / GCP 部署脚本
└── extras/              # 附加工具（翻译脚本等）
```

## RAG 完整流水线

```
文档上传 → 文本抽取 → 分块 → Embedding → 存入向量库 → 查询 → 拼 Prompt → LLM 回答
```

### 1. 文档上传

三种路径：

| 路径         | 入口                                                          | 流程                                    |
| ------------ | ------------------------------------------------------------- | --------------------------------------- |
| 文件上传     | `POST /workspace/:slug/upload-and-embed`                      | Multer → collector/hotdir → 解析 → 嵌入 |
| 先解析后嵌入 | `POST /:slug/parse` → `POST /:slug/embed-parsed-file/:fileId` | 解析存 direct-uploads → 用户触发嵌入    |
| 链接抓取     | `POST /workspace/:slug/upload-link`                           | Puppeteer 抓取 → 解析 → 嵌入            |

### 2. 文本抽取（collector 服务）

collector 是独立进程（端口 8888），按文件扩展名选择 converter：

| 文件类型                            | Converter         | 依赖                 |
| ----------------------------------- | ----------------- | -------------------- |
| `.txt` `.md` `.html` `.csv` `.json` | `asTxt.js`        | 原生                 |
| `.pdf`                              | `asPDF/index.js`  | pdf-parse            |
| `.docx`                             | `asDocx.js`       | mammoth              |
| `.pptx` `.odt` `.odp`               | `asOfficeMime.js` | officeparser         |
| `.xlsx`                             | `asXlsx.js`       | node-xlsx            |
| `.epub`                             | `asEPub.js`       | epub2                |
| `.mp3` `.wav` `.mp4`                | `asAudio.js`      | @xenova/transformers |
| `.png` `.jpg`                       | `asImage.js`      | Tesseract.js (OCR)   |
| `.mbox`                             | `asMbox.js`       | mbox-parser          |

**输出格式**：`{ id, pageContent, token_count_estimate, title, chunkSource, ... }`

Server 通过 `CollectorApi` 调用 collector，使用 `CommunicationKey` + `X-Integrity` 签名验证请求完整性。

### 3. 分块（TextSplitter）

- 文件：`server/utils/TextSplitter/index.js`
- 使用 LangChain 的 `RecursiveCharacterTextSplitter`
- `chunkSize` 由 `EmbedderEngine.embeddingMaxChunkLength` 决定
- 支持 `chunkOverlap`、`chunkHeaderMeta`、`chunkPrefix` 配置

### 4. Embedding

- 工厂函数：`getEmbeddingEngineSelection()`（`server/utils/helpers/index.js`）
- 由 `EMBEDDING_ENGINE` 环境变量选择，默认 `native`（本地 @xenova/transformers 模型）
- 接口：`embedTextInput(text)` / `embedChunks(textChunks)`

支持的引擎：

| 引擎                              | 说明                                   |
| --------------------------------- | -------------------------------------- |
| `native`                          | 本地 @xenova/transformers，零 API 调用 |
| `openai`                          | OpenAI Embedding API                   |
| `ollama`                          | 本地 Ollama                            |
| `azure`                           | Azure OpenAI                           |
| `cohere` / `voyageai`             | 第三方 Embedding                       |
| `gemini` / `mistral` / `lmstudio` | 其他                                   |

### 5. 向量存储

- 工厂函数：`getVectorDbClass()`（`server/utils/helpers/index.js`）
- 由 `VECTOR_DB` 环境变量选择，默认 `lancedb`
- 基类：`server/utils/vectorDbProviders/base.js`

核心接口：

```
connect()
addDocumentToNamespace(namespace, documentData, path)
deleteDocumentFromNamespace(namespace, docId)
performSimilaritySearch({ namespace, input, LLMConnector, similarityThreshold, topN })
```

写入流程：`TextSplitter.splitText()` → `EmbedderEngine.embedChunks()` → 写入向量库 → 记录到 `document_vectors` 表

支持的向量库：LanceDB / Pinecone / Chroma / Qdrant / Milvus / Zilliz / Weaviate / Astra / PGVector

### 6. 查询与 LLM 回答

入口：`POST /workspace/:slug/stream-chat` → `server/utils/chats/stream.js`

流程：

1. 加载 Pinned documents（固定文档直接进入 context）
2. 加载 Parsed files（`WorkspaceParsedFiles.getContextFiles()`）
3. **向量检索**：`VectorDb.performSimilaritySearch({ namespace, input, similarityThreshold, topN })`
4. `fillSourceWindow` 填充 context 窗口
5. 可选 rerank（`vectorSearchMode === "rerank"`）
6. `LLMConnector.compressMessages()` 构造 prompt
7. `LLMConnector.streamGetChatCompletion()` 流式输出

## 认证机制

### 单用户模式

- 环境变量 `AUTH_TOKEN` 设置密码
- JWT 中 `p` 字段用 `EncryptionManager` 加密
- 校验：`bcrypt.compareSync(decrypt(p), hashSync(AUTH_TOKEN))`

### 多用户模式

- `SystemSettings.isMultiUserMode()` 开启
- JWT 解析 → 查 `users` 表 → 检查 `suspended` 状态
- 角色：admin / manager / default

### API Key

- `/api/v1/*` 路由使用 `validApiKey` 中间件
- 校验 `api_keys` 表

## 抽象层设计（重点学习）

AnythingLLM 最值得学习的架构模式——**面向接口编程**：

```
环境变量 VECTOR_DB / EMBEDDING_ENGINE / LLM_PROVIDER
    ↓
工厂函数 getVectorDbClass() / getEmbeddingEngineSelection() / getLLMProvider()
    ↓
返回具体实现（都遵循统一接口）
    ↓
业务代码只调接口，不关心具体实现
```

新增一个 LLM 提供商或向量库，只需：

1. 在对应目录下新建文件夹
2. 实现统一接口
3. 在工厂函数中注册
